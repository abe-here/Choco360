# 03. Profile & Onboarding

## 對應頁面與模組
- `components/Dashboard.tsx` (總覽面板)
- `components/Profile.tsx` (個人中心)

## 總覽面板 (Dashboard)
1. **歡迎模組與權限徽章**
   - 顯示用戶的名稱歡迎詞與系統願景。
   - 右上角顯示用戶目前的權限身分 (`Manager / 主管層級` 或 `IC / 專業貢獻者`) 並帶有動態綠燈。
2. **個人指標看板 (Metrics Board)**
   - **待填寫評價**: 統計目前分配給該用戶需完成的反饋任務數量。
   - **收到回饋數**: 統計目前該用戶作為「受評人」時，已經收到且狀態為 submitted 的反饋數量。
   - **邀請函審核狀態**: 顯示該用戶自己發起的最新一筆問卷（Nomination）目前的審核狀態 (`主管審核中`, `已核准`, `已駁回`) 以及負責審核的主管名稱。
3. **智慧流程導引 (Workflow Guide)**
   - 根據用戶身份顯示不同的引導文案（主管發起全方位評鑑 vs IC 發起同儕 360）。
   - 提供「立即發起邀請」按鈕快速跳轉至發送邀請頁面。
   - 下方包含 4 步驟卡片導引：建立名單 -> 主管審核 -> 同步評鑑 -> AI 報告生成。
4. **待辦事項清單 (Action Items)**
   - 列出所有等候用戶「給予回饋」的任務（通常為其他同事發起的邀請函且已經過主管核准）。
   - 列表包含發起人的頭像 (Avatar)、名稱、問卷標題，以及「開始評鑑」按鈕。
   - 若無任務，則顯示預設的空狀態插圖與引導提示。

## 個人中心 (Profile)
1. **數位形象管理 (Avatar Management)**
   - 顯示 Avatar Seed Engine (串接 Dicebear API) 的頭像。
   - 提供「刷新隨機預覽」按鈕，允許用戶產生新的隨機頭像。
   - 修改頭像、格言或超能力後，個人資訊卡底部會顯示明顯的「**尚未儲存的變更**」提示橫幅。
   - 只有點擊「**確認儲存**」才會正式將所有變更透過 `api.updateUser()` 寫入資料庫。
2. **個性化設定 (Personalization)**
   - **人生格言 (Motto)**：使用者可在個人中心的格言欄位輸入一段文字，儲存後將顯示在「社群交流」的個人卡片中。
   - **超能力裝備 (Active Aura)**：使用者可從「已解鎖超能力」清單中，選取一個作為「當前裝備」。裝備後，頭像背後會出現對應類別顏色的靈魂光環 (Aura)，並同步影響社群廣場中的個人卡片外觀。
   - 超能力的觸發途徑：由 AI 分析 360 回饋時自動提取並寫入 `profiles.unlocked_superpowers`。
3. **個人唯讀資訊區塊 (Read-only Profile Data)**
   - 由於屬企業內部系統，核心資料無法由前台修改。
   - 包含欄位：顯示姓名、公司信箱、職位角色、所屬部門。
   - 底部顯示「直屬主管 (名單審核人)」的名稱與頭像。
4. **開發者留言板 (System Message Board)**
   - 使用者可以寫下建議或鼓勵給系統開發團隊（所有人都會看到）。
   - 具有防止空白文字、送出載入中 (`isPosting`) 的防呆機制。
   - 下方列表會按時間顯示所有歷史留言，包含留言者頭像、名稱、時間戳記與內容文本。

## 社群交流 (Community Hub) — Beta 功能
> 本功能以「開發中」狀態並行發展，不影響現有個人中心功能。
1. **英雄榜網格 (Member Grid)**：以深色卡片形式展示全體在職員工（`status !== 'resigned'`）。
2. **個人卡片 (Member Card)**：顯示員工頭像、姓名、部門與職稱、人生格言、超能力稱號（Hover 時浮現）。
3. **超能力光環 (Aura VFX)**：每個卡片背景根據員工當前裝備的超能力類別渲染對應顏色的模糊光暈，Hover 時光暈放大增強。
4. **搜尋**：支援以姓名、部門、超能力稱號進行即時篩選。
