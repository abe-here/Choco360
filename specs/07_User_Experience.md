# Choco360 使用者體驗與流程導引規格 (UX & Workflow Guide)

本文件定義使用者登入後的首頁導覽體驗與標準評鑑流程 SOP。

## 1. 登入總覽 (Dashboard Experience)

### 1.1 身份識別與 Badge
- 系統應在頂部顯眼處標示使用者目前身分：
    - **IC (Individual Contributor)**：顯示為「IC / 專業貢獻者」。
    - **Manager**：當 `isManager` 為 true 時，顯示為「Manager / 主管層級」。

### 1.2 智慧流程導引 (Smart Workflow Guide)
- **視覺呈現**：採用 Dark Mode 風格的區塊，具備進度感與專業感。
- **動態內容**：
    - **IC 版面**：標題為「發起同儕專業 360」，強調與同事間的協作補位與專業成長。
    - **主管版面**：標題為「發起全方位 360 評鑑」。系統將提示主管可根據需求選擇：
        - **Peer Review 模式**：邀請平行部門主管或專案夥伴，優化跨團隊協作。
        - **Leadership Review 模式**：邀請直屬下屬與團隊成員，檢視領導風格與影響力。

### 1.3 狀態透明度 (New)
- **審核進度透明化**：當邀請函處於 `Pending` 狀態時，Dashboard 應清晰顯示「正在等待哪位主管審核」。
- **邏輯**：透過比對 `nomination.managerId` 與使用者清單，動態抓取主管姓名顯示，以利使用者後續追蹤。

## 2. 標準評鑑生命週期 (The 4-Step SOP)

### 第一步：建立名單 (Nomination)
- **行為**：使用者進入「邀請反饋」分頁。
- **操作**：
    1. **選取問卷**：根據想獲取的反饋維度挑選對應問卷（如：管理行為問卷、專業成長問卷）。
    2. **動態名單**：
        - IC 邀請 1-8 位同儕。
        - 主管可靈活組合同儕 (Peers) 與下屬 (Subordinates)，以獲取真正的 360 度觀點。

### 第二步：主管審核 (Manager Approval)
- **行為**：系統自動發送通知給「個人中心」定義的直屬主管。
- **邏輯**：主管可針對受邀人名單進行「新增」或「移除」操作。
- **特殊情況**：若發起人本身就是 System Admin 或最高層級，其名單仍建議由一級主管或 HR 核准，以確保中立。

### 第三步：同步評鑑 (Synchronous Feedback)
- **行為**：當主管點擊「核准」後，受邀人會收到任務通知。
- **自我評鑑**：受評人本人必須完成「自評問卷」，這是 AI 分析的重要對比基準。

### 第四步：AI 報告生成 (AI Insight)
- **觸發時機**：當所有受邀人完成填寫，或週期截止時。
- **處理**：Gemini 3 Pro 解析量化分數與質性回饋（SSC 模型），生成個人發展計畫。

---
## 3. 待辦事項清單 (Action Items)
- 總覽頁面底部必須條列「目前需由我填寫」的他人邀請。
- 具備「自我評鑑」專屬標籤，區分他評任務與自評任務。
