# 05. Feedback Execution

## 對應頁面與模組
- `components/FeedbackForm.tsx`

## 使用者旅程 (User Journey)
1. 員工進入 **填寫反饋 (FeedbackForm)** 頁面。
2. 畫面載入「待處理的反饋任務」清單 (包含自評與他評)。顯示受評人頭像、名稱、評鑑標題與截止日期。
3. 點擊卡片上的「開始填寫 (Start Feedback)」按鈕，展開該同仁的問卷表單。若是自己則會顯示為「自我評鑑問卷」。
4. **填寫過程**：
   - 根據問卷設定的維度 (Dimensions) 與題目 (Questions) 進行回答。
   - **量表題 (Rating)**：提供 1 到 5 分的按鈕選項 (5: 總是如此, 4: 經常如此, 3: 偶爾如此, 2: 很少如此, 1: 不清楚)。
   - **文字題 (Text)**：提供文字輸入框，請使用者輸入具體觀察、實例描述或佐證。
5. **送出檢查**：
   - 點擊「完成填寫並送出」按鈕。
   - 系統會檢查所有**量分題**是否皆已完成評分，若有遺漏，會跳出 Alert 提示「請完成所有量分題的評比後再送出。」
   - 目前未強制要求文字題的字數及自動儲存功能 (Auto-save)。
6. **成功提交**：
   - 送出 API 成功後，畫面會切換至成功提示畫面 (顯示「反饋提交成功！」與綠色打勾圖示)。
   - 點擊「返回任務清單」按鈕，即可重新載入最新的待辦清單，方才填寫完成的任務將不再顯示。
