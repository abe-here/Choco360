# 05. Feedback Execution

## 對應頁面與模組
- `components/FeedbackForm.tsx`

## 使用者旅程 (User Journey)
1. 員工進入 **給予回饋 (Give Feedback)** 頁面。
2. 畫面左側或上方，陳列出「已被主管核准，需要我填寫的同仁名單」。
3. 點擊某位同仁的頭像，展開該同仁的問卷表單。
4. **填寫過程**：
   - 根據系統設定的維度（例如：團隊合作、專業技能、溝通表達）進行雷達圖量化給分 (1-5 分)。
   - 在每個維度底下，必須填寫「具體事例 (Comments)」，字數不可少於 20 字。
5. **自動儲存 (Auto-save)**：
   - 填寫中若不慎跳出，或重新整理網頁，已填寫內容應從 LocalStorage 或 Database Draft 撈回。
6. **送出檢查**：
   - 點擊「送出回饋」。
   - 若有必填未填（未給分、字數不足），系統亮紅字阻擋送出。
   - 成功後，該同仁移出「待辦清單」，打上綠色勾勾圖示。
