# Choco360 員工管理模組規格 (Employee Management)

本文件定義了「控制中心 > 員工管理」分頁的所有行為與視覺規範。

## 1. 組織架構標準定義 (Hierarchy)

### 1.1 部門 (Departments)
系統強制要求員工必須隸屬於以下七大部門：
- `Product`, `Data`, `Marketing`, `Content`, `HR/ADM`, `AVOD`, `Finance`

### 1.2 職位角色 (Roles)
標準職稱定義（影響權限與報表分類）：
- `PM`, `Designer`, `Developer`, `QA`, `IT`, `DA/DE`, `CS`, `Content`, `HR/ADM`, `AVOD`, `Marketing`, `Finance`

---

## 2. 員工清單 (User List View)

### 2.1 介面組成
- **全域搜尋**：支援透過「姓名」、「Email」、「部門」進行即時過濾。
- **數據表格**：
    - **員工資訊**：顯示圓角 12px 的 Avatar、粗體姓名、以及細字 Email。
    - **權限標籤**：若該用戶為 `isSystemAdmin`，姓名旁需顯示深靛藍色 (Indigo-900) 的「系統管理員」小標籤。
    - **角色與部門**：分兩行顯示，第一行顯示角色，第二行顯示灰色的部門標籤。
    - **所屬主管（問卷審核）**：顯示該員工的直屬主管姓名，此資訊決定了該員工發起的評鑑名單將由誰進行審核。
- **操作按鈕**：每行末端具備「詳情與編輯」按鈕。

---

## 3. 員工編輯器 (User Editor Drawer)

當點擊「新增員工」或「詳情與編輯」時，從螢幕右側滑出寬度約 448px (max-w-md) 的抽屜。

### 3.1 基礎資訊編輯
- **頭像預覽**：頂部顯示 96x96 的大頭像，具備邊框與陰影。
- **顯示姓名**：必填欄位。
- **公司信箱**：
    - 新增時：必填，需符合 `@choco.media` 格式。
    - 編輯時：**鎖定不可修改** (Disabled)，以確保資料庫關聯完整性。

### 3.2 組織屬性選擇
- **職位與部門**：採下拉選單 (Select)，僅限選擇 1.1 與 1.2 定義之項目。
- **系統權限 (isSystemAdmin)**：下拉選擇「一般使用者」或「系統管理員」。
- **管理身分 (isManager)**：下拉選擇「否」或「是」。設為「是」後，該員工始可作為他人的「直屬主管」。

### 3.3 隸屬關係 (Reporting Line)
- **直屬主管選單**：
    - 資料來源：僅顯示系統中 `isManager` 或 `isSystemAdmin` 為 true 的員工。
    - 影響：定義了提名時，誰會收到該員工的「審核名單」請求。

---

## 4. 帳號刪除流程 (Deletion Workflow)

為了防止誤刪導致歷史報表遺失，系統採「兩階段安全確認」：
1. **觸發**：點擊編輯器底部的「刪除此員工帳號」。
2. **警示階段**：抽屜內出現紅色警示區域，顯示警告文字。
3. **確認執行**：用戶需再次點擊「確認刪除」，始呼叫 API 執行 `DELETE` 操作。
